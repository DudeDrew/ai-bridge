# Universal AI Bridge

Platform-agnostic webhook bridge for routing content between knowledge bases (Notion, Obsidian) and AI platforms (OpenAI, Claude, Pinecone).

## Features

- ✅ **Multi-Platform Support**: OpenAI, Anthropic/Claude, Pinecone
- ✅ **Multi-Source Support**: Notion, Obsidian, custom webhooks
- ✅ **Web Dashboard**: Easy configuration via browser
- ✅ **Webhook Generation**: Automatic syncing from sources
- ✅ **Security**: API key auth, webhook signatures, rate limiting
- ✅ **Deduplication**: Avoid uploading duplicate content
- ✅ **Production Ready**: Designed for Render.com deployment

## Quick Start

### 1. Deploy to Render

1. Push this repo to GitHub
2. Import to Render.com
3. Render auto-detects `render.yaml`
4. Set API keys in Render dashboard
5. Your bridge is live!

### 2. Access Dashboard

Visit `https://your-app.onrender.com/` and enter your API key (auto-generated by Render).

### 3. Create Connections

Use the web dashboard to:
1. Create routes (where content goes)
2. Create webhooks (what triggers sync)
3. Get webhook URLs
4. Configure in Notion/Obsidian

## Supported Platforms

### Destinations (AI Platforms)
- **OpenAI**: Vector stores for GPT Assistants
- **Anthropic/Claude**: Knowledge bases with prompt caching
- **Pinecone**: Universal vector database

### Sources (Knowledge Bases)
- **Notion**: Databases and pages
- **Obsidian**: Vaults (requires Local REST API plugin)
- **Custom**: Any service that can POST JSON

## Obsidian Setup

### Option 1: Webhook Plugin
1. Install "Webhooks" community plugin in Obsidian
2. Configure webhook URL from AI Bridge dashboard
3. Choose which notes/folders to sync

### Option 2: Local REST API Plugin
1. Install "Local REST API" community plugin
2. Set `OBSIDIAN_API_URL` in environment variables
3. Bridge polls vault at configured interval
